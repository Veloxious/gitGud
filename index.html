Ready to talk Version Control?

What is Git?

Before we start slinging around BASH commands lets talk what Git is. Git is a version control system that not only allows local versioning but with the addition of Github allows remote versioning as well.

There a couple major perks to this:
  • Local commits allow a safe space to build and expirement.
  • resetting the repo to a previous commit is VERY usefull when the project breaks.
  • Remote Repos allow for colabriration with peers and the open source community.
  • The different github workflows allow for various types of teams (startup to enterprise) to develop.

Command Line Interface

Lets cover the basic CLI for git.

To initalize your Git Repo use:
git init

To create a remote repo on github from the CLI use:
hub create

To stage your modified files use:
git add .

To commit your staged files (with a message) use:
git commit -m "commit message goes here"

To push your code to github (for the first time, with a Remote Upstream) use:
git push -u origin master

Feature Branch Workflow

Branching might be the single most powerful feature of Git and Github. Branching, when used correctlly, allows for collaboration while avoiding the common pitfalls of merge conflicts.

Lets start with cloneing down a remote repo:
git clone https://github.com/Veloxious/gitGud.git

CD into the new directory:
cd gitGud

Branch out from the master branch, for this we have two options:
git branch newBranchName
git checkout newBranchName

OR

git checkout -b newBranchName

Once you branch out add, commit and push at regular interivals
